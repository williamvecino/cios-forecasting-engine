Enforce it by making decision classification a mandatory gated layer for every ingestion path, not a prompt preference.

What to make mandatory

For every imported file type — PDF, PPT, DOC, image, screenshot, pasted text, bundle — the system must always produce these in order:

Domain
Decision archetype
Primary decision
Secondary decisions
Signals found
Signals to investigate
Confidence
Why this classification was chosen

If any of the first 4 are missing, the ingestion is incomplete and cannot proceed silently.

The enforcement rule

Before the system shows a proposed decision, it must pass this internal check:

Is this a strategy problem, operational problem, access problem, competitive problem, behavior problem, or vendor-selection problem?
What evidence in the document supports that choice?
Was vendor selection explicitly requested, or is the document actually about solving a launch/business problem through an agency?

That last question is critical.

The universal architecture

For any ingestion doc, build this fixed pipeline:

Content recovery
→ Document type detection
→ Domain classification
→ Decision archetype classification
→ Primary decision generation
→ Signal extraction
→ Missing signal generation
→ Confidence scoring
→ User confirmation

Never skip archetype classification.

The key design principle

The system must distinguish between:

Document wrapper
Actual business decision

Example:

RFP is the wrapper.
Launch strategy is the decision.

Slide deck is the wrapper.
Competitive differentiation is the decision.

Clinical paper is the wrapper.
Adoption evidence relevance is the decision.

This rule generalizes across all docs.

How to build it so it generalizes
1. Add explicit evidence spans

For every classification, store the exact text snippets that support it.

Example for the Roche RFP:

“pre-launch and launch objectives”
“focus on developing a campaign for patients”
“launch a best in class anti-PDL1 mAB”
“competitive landscape”
These support Launch Readiness / Competitive Positioning, not vendor selection.
2. Add negative rules

Not just what supports a class, but what disqualifies it.

Example:
If vendor selection is not explicitly the main ask, do not classify as Vendor Selection.

3. Add confidence thresholds

If confidence is low, the system must say:

“Likely decision archetype: Launch Readiness
Alternative: Stakeholder Engagement”

and ask for confirmation.

4. Add adjudication rules

If two archetypes compete, prefer the one tied to the core business objective, not the procurement format.

The four checks every ingestion must pass
Check 1 — Wrapper vs decision

What is the file?
What is the actual business decision?

Check 2 — Explicit ask vs implied ask

What is directly requested?
What is strategically implied?

Check 3 — Primary vs secondary decisions

One main decision, others subordinate.

Check 4 — Evidence support

What text proves the classification?

Pass/fail rules

Fail if:

vendor selection is inferred without explicit support
the primary decision is generic
the classification ignores obvious business context
no evidence spans are attached
the same document yields inconsistent archetypes on rerun

The one thing that will make this stick

Make classification output persisted and auditable.

Every ingestion should save:

detected document type
detected archetype
evidence spans
confidence
alternative archetype
final user-confirmed archetype

That way the system learns and you can audit failures.

Bottom line

To enforce this across any ingestion doc:

make classification mandatory
separate wrapper from real decision
require evidence spans
lock archetype choices
add negative rules
autotest against a gold set
persist every classification decision

That is how you build the same judgment across the whole platform
