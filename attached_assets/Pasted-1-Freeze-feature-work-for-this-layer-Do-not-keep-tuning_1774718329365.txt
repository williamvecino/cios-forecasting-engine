1) Freeze feature work for this layer

Do not keep tuning signals one by one.

For this part of CIOS, lock:

current signal schema
current gate list
current question format
current Judge layout

Otherwise you will not know whether failures come from logic or moving inputs.

2) Build a Judgment Integrity Test Set

This should be separate from the ingestion gold set.

You need about 12–15 fixed forecasting cases with expected behavior.

Not just expected probability.
Expected relationship logic.

Each test case should include:

Question
Signals
Gate states
Expected directional outcome
Expected confidence range
Expected narrative constraints
Expected recommendation range

Example:

Case A

Positive signals dominate
No hard gate
One moderate barrier
Expected:
outcome cannot be “low adoption most probable”
confidence cannot be high if gates unresolved
recommendation cannot be “do not allocate significant resources”

This is how you catch contradictions, not just bad numbers.

3) Add a visible Judge debug panel

Until this exists, testing will stay vague.

For every run, show:

positive signal count
negative signal count
weighted positive total
weighted negative total
raw pre-gate forecast
each gate cap
post-gate forecast
confidence inputs
recommendation trigger
narrative source fields

That panel should let you answer in 10 seconds:

Did the system calculate correctly, or did the page drift?

4) Define pass/fail rules

You need binary integrity rules.

Examples:

State sync rule

Step 3 must use the exact latest Step 2 saved state.

Confidence rule

If unresolved contradictions or major unresolved gates exist, confidence cannot be “high” unless there is explicit low-sensitivity proof.

Headline rule

If positive weighted support exceeds negative weighted drag and no hard gate exists, the headline cannot say “barriers outweigh supporting factors” unless the cap math explicitly justifies it.

Recommendation rule

A strong negative planning recommendation cannot appear unless final probability is below threshold and the barrier structure justifies low investment.

Narrative rule

Narrative statements must map to current structured state, not cached summaries.

These are machine-testable.

5) Run three test modes

You should not test only with normal usage.

A. Static regression tests

Same exact case every run.
Should produce same result every time.

This tests deterministic behavior.

B. Sensitivity tests

Change only one variable:

one signal strength
one gate
one confidence score
one time horizon

Then verify the output changes in the expected direction and magnitude.

This tests causal sanity.

C. Contradiction tests

Intentionally create cases like:

many positives but one unresolved gate
weak evidence but strong access
equal positives and negatives

Then check whether the system explains uncertainty correctly.

This tests judgment realism.
