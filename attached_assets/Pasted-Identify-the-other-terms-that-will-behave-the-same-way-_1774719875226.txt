Step 2 — Define the universal decomposition rule

Add one rule to your judgment layer.

Rule

If a constraint is expressed as a category rather than an observable condition,
the system must enumerate its component drivers.

In code logic terms:

if (constraint.isAbstract) {
   enumerateDrivers(constraint)
}

You do not need machine learning.
You need a deterministic mapping.

Step 4 — Rank the drivers automatically

Do not leave them as a list.

Rank them by impact.

Use a simple scoring rule:

impactScore =
   severity
   × probability
   × reach

Where:

Severity
How much it slows adoption

Probability
Likelihood of occurring

Reach
How many users are affected

Then sort descending.

Step 5 — Show the ranked output in the Judge page

Instead of:

Operational readiness is holding this back

You show:

Primary constraint: Operational readiness

Drivers ranked by impact:

Prior authorization predictability — High
Site workflow integration — Moderate
Staff training — Low

Now the system becomes actionable.

Step 6 — Add one enforcement rule

This prevents vague statements from ever appearing again.

Enforcement rule

No abstract constraint may be displayed without at least one ranked driver.

In code terms:

if (constraint.drivers.length === 0) {
   throw Error("Abstract constraint without drivers")
}

That single rule eliminates the problem permanently.
