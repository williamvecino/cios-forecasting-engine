CIOS NEXT HARDENING PHASE

The new bucket calibration, question intelligence, and challenge mode are live.
Now the objective is not feature expansion. It is validation hardening and calibration discipline.

Build the next phase in this order:

1. CALIBRATION DIAGNOSTICS PANEL
Add a structured calibration diagnostics view showing:
- bucket
- sample size
- mean signed error
- mean absolute error
- correction applied
- last updated date
- whether correction is active or below threshold

Also show:
- pre-calibration probability
- post-calibration probability
- absolute delta applied

Goal:
Make calibration behavior fully inspectable.

2. CALIBRATION GUARDRAILS
Add guardrails so bucket corrections cannot become unstable.
Implement:
- max absolute bucket adjustment cap
- recency-weighted averaging option for correction updates
- warning flag if bucket sample size is low
- warning flag if correction direction flips frequently

Goal:
Prevent noisy small-sample correction behavior.

3. FORECAST TRACE MODE
Add a structured trace output for each forecast:
- top positive drivers
- top negative drivers
- actor bottleneck
- analog support summary
- calibration layer summary
- fragile assumption

This should be a structured audit panel, not prose.

Goal:
Let analyst see exactly why the forecast is what it is.

4. VALIDATION ROUND 2
Run a stricter validation focused on calibration stability:
- at least 2 cases in moderate range
- at least 2 cases in high-confidence range
- at least 1 psychiatry case
- at least 1 cardiology case
- compare raw vs calibrated vs actual
- check whether moderate cases remain stable
- check whether high-confidence overforecasting is reduced
- summarize whether calibration is improving or becoming noisy

Before building, show:
1. architecture changes
2. files to modify
3. data model changes
4. validation plan

1. diagnostics panel
