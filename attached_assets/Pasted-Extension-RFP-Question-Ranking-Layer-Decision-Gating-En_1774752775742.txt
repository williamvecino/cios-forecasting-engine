Extension — RFP Question Ranking Layer (Decision Gating Enhancement)

This is now a mandatory sub-step inside Decision Gating, not a UI feature.

It determines which strategic question becomes the primary CIOS case.

Step 1A — RFP Question Ranking (inside Decision Gating)
Purpose

Identify the core competitive or strategic question and rank the top candidates.

This prevents:

vague starting questions
misaligned cases
scattered analysis
Inputs

RFP document
Brand
Indication
Competitive context

Processing Logic

The agent must:

Identify the central strategic decision
Generate candidate decision questions
Rank them by strategic importance
Provide a rationale for ranking
Required Output

Exactly 2–3 ranked questions.

No more.
No less.

Server-Side Validation Rules

These must run automatically.

If AI output fails, backend must correct.

Rule 1 — Rank Presence

Every question must have:

Rank

If missing:

Assign rank automatically.

Rule 2 — Rank Uniqueness

Ranks must be:

1
2
3

No duplicates.

Rule 3 — Question Count

Allowed:

2 or 3

If more:

Keep top 3.

If fewer:

Generate additional candidate.

Rule 4 — Strategic Focus

The top-ranked question must be:

Competitive
Adoption
Launch
Positioning

Not:

Operational
Administrative
Vendor selection

Step 1C — Saved Questions for Later

This is a persistence mechanism.

It prevents lost strategic questions.

Trigger

Any unselected ranked question is automatically saved.

Why This Layer Matters

It solves three systemic risks.

Risk 1 — Wrong Starting Question

Without ranking:

The system may analyze the wrong problem.

Ranking fixes that.

Risk 2 — Question Drift

Without saved questions:

Important decisions disappear.

Saved Questions fixes that.

Risk 3 — Fragmented Strategy

Without prioritization:

Teams analyze everything.

Ranking focuses attention.

Integration Rules — Non-Negotiable

These protect system stability.

Rule 1

Ranking must happen before MIOS.

Rule 2

Only one question runs per case.

Rule 3

Saved questions must not trigger analysis automatically.

Rule 4

Ranking logic must be deterministic.

Validation Test for This Feature

Use the Roche anti-PDL1 RFP.

Expected result:

Rank 1

Rank 3

Failure conditions:

More than 3 questions

Operational questions ranked first
