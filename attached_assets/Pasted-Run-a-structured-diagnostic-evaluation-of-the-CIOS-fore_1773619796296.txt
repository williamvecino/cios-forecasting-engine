Run a structured diagnostic evaluation of the CIOS forecasting engine.

Do not modify the system yet. First analyze and report.

Evaluate the system across the following dimensions and produce a diagnosis report.

1. Forecast Mechanics
Verify the Bayesian pipeline is functioning correctly:
- Prior → signal likelihood ratios → actor adjustment → posterior probability
Test whether changing signals materially changes the posterior probability.
Report whether the model behaves as a true Bayesian update system or if it is mostly static.

2. Clinical Realism
Evaluate whether signal effects align with realistic HCP behavior.
Example tests:
- strong RCT superiority
- guideline inclusion
- payer restriction
- safety signal
Report whether forecast direction matches expected clinical adoption logic.

3. Actor Dynamics
Assess whether actors respond differently to signals.
Actors to evaluate:
- academic specialists
- community physicians
- subspecialists
- payers
- hospital systems
- guideline committees
Report whether actor effects are independent or overly uniform.

4. Analog Engine Capability
Evaluate the Analog Matches module.
Check whether the engine:
- retrieves relevant historical cases
- ranks them by similarity
- explains similarity
Report whether analog reasoning is currently functional or mostly placeholder.

5. Signal Weighting
Analyze whether signal likelihood ratios vary meaningfully by signal type.
Identify whether signals currently have overly uniform LR values.
Report whether signal weighting reflects realistic adoption drivers.

6. Signal Sensitivity
Run scenario tests:
Scenario A:
- strong clinical evidence
- strong guideline support
- high cost barrier

Scenario B:
- moderate clinical evidence
- strong KOL advocacy
- moderate payer friction

Compare posterior outputs and explain differences.
Report whether the model meaningfully differentiates scenarios.

7. Forecast Explainability
Assess whether each forecast clearly communicates:
- key drivers
- key constraints
- actor reactions
- sensitivity factors
Report whether explanations are transparent enough for strategic decision use.

8. Calibration Infrastructure
Evaluate the calibration module.
Confirm that the system:
- logs forecasts
- allows outcome recording
- computes Brier score and mean error
Report whether calibration infrastructure is functional or only partially implemented.

9. System Intelligence Level
Assess whether the engine behaves more like:
A) a UI wrapper around simple probability math
B) a structured forecasting engine
C) an Aaru-style reasoning engine

Explain why.

10. Improvement Priorities
After completing the analysis, identify the top five improvements that would most increase the system's forecasting intelligence.

Output format:
Provide a structured diagnosis including:
- strengths
- weaknesses
- missing capabilities
- immediate improvements
- long-term architecture recommendations
