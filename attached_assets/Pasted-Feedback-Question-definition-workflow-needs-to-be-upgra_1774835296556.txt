Feedback: Question-definition workflow needs to be upgraded from passive input to AI-structured input.

Right now the system lets the user write a question and move forward too quickly. That creates downstream instability in outcomes, signals, and judgment language. We need a stricter workflow.

Required new workflow

User enters a draft question in plain language.
AI proposes a structured forecast question.
User can accept or edit it.
If the user edits it, AI must evaluate whether the edited version is feasible to model.
AI must then either:
approve it,
refine it,
or reject it as not feasible and explain why.
AI must explain why its proposed wording is better: measurable outcome, clear time horizon, observable event, modelable causal link.
After the question is finalized, AI must recommend whether the outcome structure should be binary or multi-state.

What AI should validate

Every final question should be checked for:

clear outcome,
explicit time horizon,
observable/measurable event,
decision relevance,
feasibility for the model.

Important rule

AI should not just rewrite the question. It must explain why the rewritten version is the correct forecasting form.

Example using the current case

User intent: FDA action on Xarelto due to GI bleeding risk.

AI should propose something like:
“Will the FDA implement a safety-related label change or safety communication for Xarelto within the next 12 months due to GI bleeding risk signals?”

Then AI should explain:

“regulatory action” alone is too broad,
“label change” and “safety communication” are observable,
the 12-month horizon is plausible,
the outcome is decision-relevant and modelable.

Binary vs multi-state outcomes

We agreed not all cases should be binary.

System rule:

Use binary only when the outcome is truly yes/no.
Use multi-state when the outcome can occur in staged or graded forms.

Examples

Binary is appropriate for:

approval vs no approval,
restriction imposed vs not imposed.

Multi-state is appropriate for:

no action,
safety communication,
label update,
restricted indication.

For the current regulatory case, AI should recommend multi-state as the better structure, because FDA response is often staged rather than binary.

UI changes needed

Add a “AI Suggested Question” box above the final accepted question.

Add a “Feasibility Check” box after user edits, with one of:

Feasible
Feasible with refinement
Not feasible

Add a short explanation box:
“Why this is the right forecasting structure.”

Add an “Outcome Structure Recommendation” box:

Binary
Multi-state

If multi-state is recommended, AI should pre-populate the proposed outcome states.

Core product principle

Question structure should be AI-assisted, user-confirmed, and model-validated before the user reaches comparison groups. This should become mandatory, because bad question framing is causing instability in everything downstream.
