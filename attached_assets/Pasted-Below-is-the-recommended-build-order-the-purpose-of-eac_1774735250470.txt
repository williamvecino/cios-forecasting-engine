2. What each agent does and where it lives
Define Question
A. Decision Gating Agent

Lives in: Define Question
Purpose: Interpret the uploaded brief / RFP / PDF and identify the real business decision.
Output: primary decision, secondary decisions, document type, relevant spans, missing information, downstream routing.

B. Question Structuring Agent

Lives in: Define Question
Purpose: Convert broad user text or raw document asks into 1–3 bounded decision questions.
Output: active question, optional sub-questions, question type, horizon, target outcome.

Add Information
C. External Signal Scout

Lives in: Add Information
Purpose: Find relevant external signals for the active case.
Output: candidate external signals with source, freshness, confidence, suggested direction/strength.

D. Signal Normalizer / Deduplicator

Lives in: Add Information
Purpose: Merge all signal sources into one clean register and prevent overlap/double counting.
Output: normalized signal register.

E. Assumption Registry Agent

Lives in: Add Information + Diagnostics
Purpose: Detect explicit and implicit assumptions in the case.
Output: assumption list, sensitivity flags, confidence level.

F. MIOS Adapter / MIOS Signal Agent

Lives in: Add Information
Purpose: Convert MIOS outputs into CIOS-ready evidence signals.
Output: evidence-derived signals, vulnerabilities, strength map.

G. BAOS Adapter / BAOS Signal Agent

Lives in: Add Information
Purpose: Convert BAOS outputs into CIOS-ready behavioral signals.
Output: objection signals, behavior-friction signals, stakeholder resistance signals.

H. Import Adapter Agent

Lives in: Add Information
Purpose: Read workbook or structured import and populate signals without changing engine logic.
Output: imported structured signals.

I. Provenance / Traceability Agent

Lives in: Add Information + Judge
Purpose: Preserve source origin for every signal.
Output: hidden and user-visible provenance metadata.

J. Signal Quality Agent

Lives in: Add Information
Purpose: Score reliability, freshness, directness, and duplication risk of signals.
Output: quality score and warnings.

K. Conflict Resolver Agent

Lives in: Add Information
Purpose: Detect and manage conflicting signals across sources.
Output: merged, flagged, or separated conflicts.

Judge
L. Core CIOS Judgment Engine

Lives in: Judge
Purpose: Produce the forecast, confidence, top drivers, and top constraints.

M. Case Comparator / Prior Structuring Agent

Lives in: Judge
Purpose: Compare current case to internal/external prior cases and structure analog logic.
Output: analogs, priors, divergence analysis.

N. Constraint Decomposition Agent

Lives in: Judge
Purpose: Break vague constraints into ranked concrete drivers.
Output: ranked sub-drivers under each constraint.

O. Explanation / Ask CIOS Agent

Lives in: Judge
Purpose: Answer “why,” “what if,” “how to fix,” and “what does this mean” questions about the current case.
Output: traceable explanations grounded in current state.

P. Integrity / Consistency Agent

Lives in: Judge + Decide
Purpose: Check that signals, confidence, narrative, recommendation, and constraints are coherent.
Output: pass/fail integrity checks and warnings.

Decide
Q. Decision Translation Agent

Lives in: Decide
Purpose: Convert forecast into a specific recommended decision and rationale.
Output: recommended action, rationale, success conditions, kill criteria.

R. Prioritization Agent

Lives in: Decide
Purpose: Rank actions by leverage and urgency.
Output: ranked action plan.

Simulate
S. Actor Segmentation Agent

Lives in: Simulate, feeds Judge
Purpose: Identify the market actors that drive adoption and define their behavioral logic.
Output: actor segments, constraints, triggers, influence weights, timing.

T. Simulation Agent

Lives in: Simulate
Purpose: Generate scenario paths from the current judgment state.
Output: best/base/worst pathways, catalyst paths, barrier-resolution paths.

U. Stakeholder Reaction Agent

Lives in: Simulate
Purpose: Simulate how each market actor reacts to changes over time.
Output: actor-specific scenario behavior.

Respond / Learning
V. Response Composer

Lives in: Respond
Purpose: Translate the decision into an output format such as strategy memo, RFP response, pitch answer, or executive summary.

W. Forecast Ledger Agent

Lives in: cross-system, visible in Library / Diagnostics
Purpose: Store forecast history, updates, and deltas.

X. Calibration Agent

Lives in: Diagnostics / Library
Purpose: Compare forecast vs reality and improve system reliability.

Y. Human Override / Editorial Agent

Lives in: cross-system
Purpose: Allow expert intervention without breaking structure.

4. Paste this into Replit

We are building CIOS as a bounded-agent decision system under the existing visible workflow:

Define Question → Add Information → Judge → Decide → Respond → Simulate

Do not add new visible steps.
Agents must live behind these steps and produce structured outputs only.

Canonical architecture rules
Each agent must have a single clear function.
Each agent must have fixed inputs and fixed outputs.
Agents must be deterministic for the same input.
No agent should perform another agent’s job.
The Core CIOS Judgment Engine remains the judge.
All new agents are support layers, not replacements for the core engine.
The system must fail gracefully when an agent cannot complete its task.
Raw uploaded documents should not flow directly into judgment if a gating agent exists.
Every signal entering judgment must preserve provenance.
Do not destabilize the current engine or UI workflow.
Build priority
Phase 1 — highest priority
1. Decision Gating Agent

Lives in: Define Question

Purpose:
Read uploaded briefs, PDFs, RFPs, decks, screenshots, and classify the real business decision.

Must output:

document type
primary decision
secondary decisions
relevant content spans
irrelevant/admin content to ignore
recommended downstream routing to MIOS / BAOS / CIOS
missing information

Rules:

distinguish wrapper from decision
ignore legal/procurement/admin boilerplate unless decision-relevant
deterministic for the same document
2. Question Structuring Agent

Lives in: Define Question

Purpose:
Convert broad or messy asks into 1–3 bounded decision questions.

Must output:

active question
optional supporting questions
question archetype
horizon
target outcome

Rules:

reject or split overly broad questions
do not let one giant multi-part strategy question become the active case
3. External Signal Scout

Lives in: Add Information

Purpose:
Find relevant outside information for the active case and convert it into candidate signals.

Must output for each candidate:

signal label
source
source date / freshness
signal type
suggested direction
suggested strength
suggested confidence
relevance score
why it matters

Rules:

do not forecast
do not inject directly into judgment
show as suggested signals first
user must accept, edit, or reject
4. Signal Normalizer / Deduplicator

Lives in: Add Information

Purpose:
Merge manual, MIOS, BAOS, workbook, and external signals into one clean signal register.

Must do:

normalize direction / strength / confidence
deduplicate overlapping signals
prevent double counting
preserve source provenance
merge or flag conflicts

Rules:

this becomes the only signal set used by the judge
5. Import Adapter Agent

Lives in: Add Information

Purpose:
Read structured workbook imports, especially MIOS / BAOS exports, and convert them into normalized signals.

Rules:

workbook is an input adapter, not a new engine
current integration must read only CIOS-ready export rows
do not let raw evidence sheets flow into judgment
6. Provenance / Traceability Agent

Lives in: Add Information + Judge

Purpose:
Store and display where each signal came from.

Each signal must retain:

source layer
source reference
originating file/workbook
if available: bridge / statement / evidence IDs

Rules:

clicking a signal should show provenance metadata
this is required for trust and explainability
7. Case Comparator / Prior Structuring Agent

Lives in: Judge

Purpose:
Compare the current case to relevant internal and external comparable cases and structure priors / analogs.

Must output:

comparable case name
brand
company
context
similarity
difference
implication
confidence

Rules:

analogs must be identifiable
do not render analogs without brand and company
this agent supports judgment but does not replace the core engine
8. Explanation / Ask CIOS Agent

Lives in: Judge

Purpose:
Answer open questions about the current case.

Supported question types:

Why is the forecast this value?
Why is this the primary constraint?
What does this term mean here?
What happens if this signal changes?
How do I solve this issue?
Why is this analog relevant?

Rules:

answers must be grounded in current case state, audit trail, signals, constraints, and analogs
no freewheeling generic chat
if not answerable from current data, say so clearly
Phase 2 — next priority
9. Actor Segmentation Agent

Lives in: Simulate, feeds Judge

Purpose:
Model real market actors and their behavioral rules.

Must output:

segment name
role in the market
behavioral characteristics
primary constraint
adoption trigger
influence weight
likely adoption timing
sensitivity to key signals
interactions with other actors

Rules:

this is not communication packaging
this is behavioral market segmentation
examples: academic HCPs, community HCPs, payers, patients, KOLs, IDNs
10. Simulation Agent

Lives in: Simulate

Purpose:
Turn the current forecast state into scenario paths.

Must output:

base path
upside path
downside path
catalyst triggers
barrier resolution points
timing effects

Rules:

simulation depends on current judgment and actor segments
do not simulate from raw inputs alone
11. Decision Translation Agent

Lives in: Decide

Purpose:
Turn judgment into a decision recommendation.

Must output:

recommended action
rationale
success conditions
kill criteria / falsifiers
12. Prioritization Agent

Lives in: Decide

Purpose:
Rank actions by leverage and urgency.

Must output:

top 3–5 actions
leverage ranking
immediate vs later actions
Phase 3 — learning and optimization
13. Forecast Ledger Agent

Lives in: Diagnostics / Library

Purpose:
Store forecast history and updates.

Must output:

prior forecast
updated forecast
delta
why it changed

Lives in: Diagnostics / Library

Purpose:
Compare forecast vs real outcomes and detect systematic bias.

Must output:

overconfidence / underconfidence flags
