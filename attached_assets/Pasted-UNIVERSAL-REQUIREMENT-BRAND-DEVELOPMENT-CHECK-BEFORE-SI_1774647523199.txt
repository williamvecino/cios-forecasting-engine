UNIVERSAL REQUIREMENT: BRAND DEVELOPMENT CHECK BEFORE SIGNAL GENERATION

For any case containing a named brand, asset, molecule, or company-linked therapy, the system must perform a live Brand Development Check before generating signals.

PURPOSE
Ensure Step 2 Add Information starts with the latest real, externally verified brand developments rather than generic placeholder signals.

REQUIRED SOURCE PRIORITY
For named brands/assets, search in this order:

1. Official company investor / press release site
2. Official brand website
3. ClinicalTrials.gov
4. Official congress / company presentation pages
5. Other high-quality public sources only if needed

RULES
1. If the brand is recognized, the system must attempt external lookup before generating generic signals.
2. Latest dated brand developments must be converted into structured signals first.
3. Generic signals may be added only after observed brand developments are captured.
4. Distinguish:
   - Observed brand developments
   - Derived implications
   - Open uncertainties
5. If external lookup fails, show:
   "No recent verified brand developments found"
   rather than inventing generic specifics.

EXAMPLE FOR ARIKAYCE
The system should detect recent official developments including:
- positive topline ENCORE Phase 3b results
- significant endpoint results
- no new safety signals
- planned sNDA filing in 2H 2026

These should become structured signals with:
- title
- source URL
- source type
- date
- direction
- impact
- reliability
- implication

DATA MODEL
Add fields to signals:
- signal_class: observed | derived | uncertainty
- source_url
- source_type
- observed_date
- citation_excerpt
- brand_verified: true | false

UI REQUIREMENT
In Step 2, add a top section:
"Latest Verified Brand Developments"

Display 1–5 most recent verified items before the rest of the signal list.

FORECAST LOGIC RULE
Observed brand developments should carry greater weight than generic inferred signals.

FAIL CONDITION
If a named brand exists and the system displays generic signals without attempting a brand development check, treat this as a logic failure.

TEST CASE
Question:
"Will ID, pulmonologists differ in adoption for ARIKAYCE within 12 months?"

Expected Step 2:
The top signal section must include the March 23, 2026 official ENCORE topline result from Insmed and its implications before any generic payer/guideline/competition placeholder signals.
