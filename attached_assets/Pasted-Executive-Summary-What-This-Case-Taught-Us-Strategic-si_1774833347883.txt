Executive Summary — What This Case Taught Us

Strategic signal:
The system correctly identified that regulatory uncertainty — not clinical evidence — is the primary constraint on near-term outcomes.

Operational signal:
The workflow logic is sound, but several UI patterns create friction, misinterpretation risk, and visual overload — especially in the simulation and assumptions layers.

Design signal:
Most problems are not analytical errors. They are:

hierarchy failures
density overload
inconsistent terminology
misplaced components
Section 1 — Question Framing (Define Step)
What Worked

The strategic question was well-formed:

Will safety concerns about increased GI bleeding with Xarelto lead to regulatory restrictions within 12 months?

This has:

clear event definition
clear comparator
explicit time horizon
decision relevance

That structure should remain the standard template.

Learning

Binary outcome framing works well for regulatory scenarios.

But the system needs stronger guardrails to prevent wording drift later in the workflow.

Example drift observed:

Outcome changed from:

Regulatory restriction

to:

Approval delay

These are not equivalent.

Fix

Lock the outcome definition across steps.

Implementation

Add:

Outcome anchor

Displayed at the top of every step:

Outcome being forecast: Regulatory restriction within 12 months

Section 2 — Comparison Groups (Define Comparison)
What Worked

Two scenarios were sufficient:

Regulatory restrictions imposed
No regulatory restrictions imposed

No changes needed.

Learning

Users do not struggle with scenarios.

They struggle with terminology.

Fix

Rename:

Scenario comparison groups

to:

Possible outcomes

Section 3 — Signals and Evidence (Add Information)
What Worked

Signal directionality was correct:

Safety signal — moderate
Regulatory review — high
Meta-analysis — stabilizing

The signal set was balanced.

Learning

Users understand signals.

But they cannot easily see which signals matter most.

Fix

Add:

Primary driver label

Example:

Primary driver: FDA safety review

Section 4 — Judgment and Forecast
What Worked

The model correctly identified:

Low near-term restriction probability
Operational constraints as primary drivers

This matches real regulatory behavior.

Learning

Probability logic was sound.

Language framing was inconsistent.

Observed issue

Executive judgment text:

A delay is more likely than on-time approval

This does not match the question.

Fix

Force semantic alignment between:

Question
Outcome
Judgment

Implementation rule

Judgment must reference the defined outcome.

Section 5 — Decision Layer (Barrier and Action Mapping)
What Worked

Barrier decomposition worked extremely well.

The system correctly identified:

Primary constraint — regulatory review
Secondary constraint — public pressure
Tertiary constraint — feasibility

This hierarchy is accurate.

Learning

The logic is strong.

But terminology creates friction.

Observed issues

Barrier language was overly technical:

Barrier decomposition

Feasibility gate

Execution gap

Fix

Use operational language.

Replace

Barrier decomposition

With:

What is blocking progress

Replace

Execution gap

With:

Constraint gap

Section 6 — Respond Layer (Strategy Output)
What Worked

The structure was excellent:

Strategic recommendation
Why this matters
Priority actions
Success measures
Execution focus

This format is executive-ready.

Learning

Message tone matters as much as logic.

Observed issues

Language was too absolute:

Do not waste resources on outreach

This reads as directive rather than strategic.

Fix

Use prioritization language.

Replace

Do not waste resources

With:

Prioritize resources

Section 7 — Simulation Layer

This is where the most serious UI issues occurred — including the ones you flagged as horrendous.

The Horrendous Issues — Root Causes

These are not minor cosmetic problems.
They affect usability and trust.

Issue 1 — The Sidebar Overload Problem

The right-side panel attempted to display:

Actor profile
Behaviors
Constraints
Triggers
Signal sensitivity
Interactions
System dynamics

All inside a narrow column.

Why This Failed

It violated a basic interface rule:

Complex logic should not live in a sidebar.

Fix

Move actor and system logic into the main content area.

Issue 2 — Three Micro-Columns in a Narrow Space

You displayed:

Behaviors | Constraints | Triggers

in parallel columns.

Why This Failed

Users cannot scan horizontally in narrow layouts.

The result:

Tiny text
Fragmented reading
Visual fatigue

Fix

Stack vertically.

Replace

Three-column layout

With:

Single vertical sequence

Behavior
Constraints
Triggers

Issue 3 — System Dynamics Section Was Unreadable

This was the most severe problem.

Current structure:

Drivers
Bottlenecks
Cascade risks

Displayed as compressed lists.

Why This Failed

The system tried to show relationships without structure.

Users need causal flow.

Fix

Convert to causal chain format.

Replace

Driver
Bottleneck
Cascade risk

With:

Driver → Constraint → Outcome

Example:

Safety signal detected
→ Regulatory review delay
→ Restricted formulary access

Issue 4 — No Hierarchy in Actor Cards

Everything was visible at once.

Weight
Timing
Behaviors
Constraints
Sensitivity
Interactions

Why This Failed

No visual priority.

Everything felt equally important.

Fix

Use progressive disclosure.

Layer 1 — Summary

Actor
Weight
Primary trigger

Layer 2 — Details (expandable)

Behaviors
Constraints
Interactions

Issue 5 — Inconsistent Color Logic

Colors were used without defined meaning.

Observed:

Red
Yellow
Green
Blue

Used inconsistently.

Fix

Assign fixed semantic meaning.

Red — constraint
Yellow — sensitivity
Green — driver
Blue — interaction

Issue 6 — Poor Spatial Balance

Large empty space on left
Dense compression on right

Why This Failed

Creates visual instability.

Fix

Use symmetric layout.

Main content width:

60–70%

Sidebar width:

30–40%

Issue 7 — “Risk Gatekeepers — No archetype assigned”

This created confusion.

Why

Gatekeepers are not adopters.

They are decision authorities.

Fix

Rename category.

Replace:

No archetype assigned

With:

Decision authority role

Issue 8 — Guidance Panel Placement

Guidance appeared below the fold.

Why This Failed

Instructions must appear before action.

Fix

Move guidance above upload field.

Issue 9 — Segment Selection Without Direction

All segments appeared equally important.

Why This Failed

Users lacked decision guidance.

Fix

Add:

Recommended segment

Example:

Recommended: Persuadables

Issue 10 — Upload Field Without Context

Users were unsure what to upload.

Fix

Add examples.

Upload material to test

Examples:

Safety letter
Guideline update
New study results
Risk mitigation plan

Cross-Cutting Design Learnings

These apply across the entire workflow.

Learning 1 — Density Is the Core Usability Risk

Most problems were caused by:

Too much information
Displayed simultaneously

Not by incorrect logic.

Learning 2 — Terminology Must Be Operational

Users respond better to:

What is blocking progress

Than:

Barrier decomposition

Learning 3 — Hierarchy Matters More Than Content

Clear structure reduces cognitive load more than reducing information.

Learning 4 — Sidebars Should Contain Navigation, Not Analysis

Complex logic belongs in the main workspace.

Priority Fix List

Implement these in order.

Priority 1 — Layout

Move actor and system logic into main content area

Priority 2 — Structure

Replace multi-column micro-layouts with vertical stacking

Priority 3 — Hierarchy

Introduce progressive disclosure for actor details

Priority 4 — Language

Standardize terminology across steps

Priority 5 — Visual Semantics

Assign fixed meaning to colors
