Good. The Calibration / Performance Dashboard is accepted as complete.

## Decision

Proceed to the next canonical build step: **Adoption Segmentation Panel**.

## Purpose

Translate forecast outputs into a segmented adoption map showing **who is likely to move first, who is lagging, what is constraining each segment, and where effort should be focused**.

This is not a generic target list. It is a structured segmentation layer tied to forecast logic.

## What this panel must answer

At minimum:

* Which segments are most likely to adopt first?
* Which segments are positive in principle but blocked operationally?
* Which segments are resistant and why?
* Where is the forecast strongest vs weakest across segments?
* Which segment should be prioritized first for action?

## Build requirements

### 1) Define segmentation schema

Create a segmentation structure that can support at minimum:

* `SegmentID`
* `SegmentName`
* `SegmentType`
* `AdoptionLikelihood`
* `ConfidenceLevel`
* `EvidenceDiversityScore`
* `PosteriorFragilityScore`
* `PrimaryDrivers`
* `PrimaryBarriers`
* `OperationalConstraints`
* `AccessConstraints`
* `BehavioralSignals`
* `ForecastHorizon`
* `PriorityRank`
* `RationaleSummary`

### 2) Initial segment categories

Support a practical first-pass set of pharma-relevant segments, expandable later.

Examples:

* Academic KOLs
* Community high adopters
* Community cautious adopters
* Access-constrained accounts
* Workflow-sensitive clinicians
* Guideline-led adopters
* Economics-sensitive decision-makers
* Competitive incumbency defenders

This does not need to be exhaustive yet, but it must support segment-level reasoning.

### 3) Segment-specific forecast translation

The panel should not create a brand-new inference engine. It should translate existing forecast logic into segment-level views using:

* signal mix
* access/workflow constraints
* KOL/community differences
* evidence translation strength
* competitive and operational context

### 4) Output required for each segment

For each segment show:

* adoption likelihood
* confidence
* top 2–4 drivers
* top 2–4 barriers
* what would most likely move the segment upward
* what is currently preventing movement

### 5) Priority logic

Add a simple priority framework so the panel can distinguish:

* high-likelihood / high-value early movers
* persuadable but blocked segments
* low-probability / low-near-term-return segments

I want the system to help answer:
**Where should effort go first?**

### 6) UI requirements

Need a usable panel that shows:

* ranked segment list
* adoption likelihood by segment
* confidence / fragility indicators
* key barriers by segment
* drilldown for each segment rationale

Functional first, polished later.

### 7) Important guardrail

This panel must remain tied to the forecast architecture.

Do not let segmentation become a generic persona exercise or marketing taxonomy layer.

The panel must stay grounded in:

* forecast outputs
* dependency-controlled signals
* reference patterns where relevant
* explicit drivers and barriers

## Deliverables

Return with:

1. segmentation schema
2. initial segment set
3. ranked segment outputs working
4. segment-level driver / barrier summaries
5. priority logic working
6. usable panel UI
7. clear note on how segment views are derived from existing forecast logic
