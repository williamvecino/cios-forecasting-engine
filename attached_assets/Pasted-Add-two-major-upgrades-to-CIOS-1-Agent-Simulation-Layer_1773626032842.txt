Add two major upgrades to CIOS:

1. Agent Simulation Layer
2. Large-Scale Learning Layer

Goal:
Move CIOS closer to a true strategic intelligence engine for forecasting HCP adoption and behavior.

PART 1 — AGENT SIMULATION LAYER

Build a modular agent simulation system that models how different stakeholders react to signals and influence adoption.

At minimum include these agents:
- academic specialists
- community specialists
- inpatient / hospital prescribers
- payers / access decision-makers
- guideline / society bodies
- competitors
- commercial field force / MSLs

For each agent define:
- core motivations
- key decision drivers
- sensitivity to evidence
- sensitivity to safety concerns
- sensitivity to convenience / delivery advantages
- sensitivity to access / reimbursement barriers
- inertia / resistance to change
- response speed
- influence on broader adoption

The system should simulate:
- which agents are early supporters
- which are resistant
- which signals affect each agent most
- how agent reactions combine into the forecast

Output should show:
- agent-by-agent stance
- adoption path by stakeholder group
- likely sequence of market movement

Important:
This should not be a static label system. It should be a structured reaction model.

PART 2 — LARGE-SCALE LEARNING LAYER

Build a learning layer so CIOS improves as more cases are added.

Requirements:
- expand the case library into a reusable memory system
- store historical cases, signals, actor reactions, and actual outcomes
- retrieve similar prior cases when a new case is created
- compare the active case to historical patterns
- track which signals were most predictive
- track where past forecasts were right or wrong
- use prior cases to inform current forecast reasoning

Add these capabilities:
1. Case memory
   - each case stores asset, disease, specialty, signal mix, actor pattern, and outcome

2. Analog retrieval
   - retrieve and rank comparable historical cases
   - explain why they are similar
   - show key differences

3. Outcome learning
   - when actual results are recorded, store them
   - compare forecast vs realized result
   - update the case memory for future use

4. Pattern summaries
   - show repeated patterns such as:
     - strong congress data but slow payer uptake
     - specialist enthusiasm but community lag
     - positive efficacy offset by safety narrative
     - convenience advantage driving adoption in specialist-heavy markets

5. Forecast support
   - when a new forecast is created, show:
     - similar past cases
     - what happened in those cases
     - what lessons should inform the current forecast

UI / workflow:
Case created
→ signals detected
→ agents react
→ similar past cases retrieved
→ forecast generated
→ outcome later recorded
→ system learns from the case

Important design rule:
Do not reduce this to a chatbot summary.
Keep it as a structured intelligence workflow inside the current CIOS architecture.

Output after implementation:
- architecture for agent simulation
- architecture for large-scale learning
- data model changes required
- UI flow changes
- example using one active case and 3 prior analog cases
