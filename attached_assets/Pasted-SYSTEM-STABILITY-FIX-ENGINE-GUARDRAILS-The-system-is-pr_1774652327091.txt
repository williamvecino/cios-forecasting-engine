SYSTEM STABILITY FIX — ENGINE GUARDRAILS

The system is producing logically inconsistent forecasts (example: 98% probability despite unresolved gating constraints).

The Bayesian core is likely functioning, but translation between UI drivers and engine inputs is not constrained.

We need to enforce deterministic guardrails at the engine boundary.

Implement the following changes.

--------------------------------

1) DRIVER DEDUPLICATION RULE

Before calculating probability:

- check for duplicate drivers
- identify duplicates by driver_id OR identical description

If duplicates exist:

- keep the highest reliability instance
- discard the rest

Log:

duplicate_driver_detected
duplicate_driver_removed

--------------------------------

2) MAXIMUM IMPACT CAP

No single driver may move probability beyond a defined limit.

Set:

MAX_SINGLE_DRIVER_SHIFT = 15 percentage points

If calculated shift exceeds limit:

cap the shift
log:

driver_shift_capped

--------------------------------

3) TOTAL SHIFT NORMALIZATION

After summing all drivers:

If total shift exceeds:

MAX_TOTAL_SHIFT = 40 percentage points

then:

normalize all shifts proportionally

log:

total_shift_normalized

--------------------------------

4) EVENT GATING CONSTRAINT

If any required event gate is unresolved or weak:

limit final probability.

Required gates:

- line_of_therapy_applicability
- stakeholder_applicability
- time_horizon_feasibility
- threshold_attainment

Rule:

If any gate == weak OR unresolved:

FINAL_PROBABILITY cannot exceed 70%

Log:

probability_limited_by_gate

--------------------------------

5) RELEVANCE CHECK INTEGRATION

If relevance panel indicates:

"signal does not directly answer the question"

then:

reduce that driver's impact by 50%

Do not allow full weighting.

Log:

relevance_penalty_applied

--------------------------------

6) RECALCULATION CONSISTENCY

Every recalculation must produce identical results if inputs are unchanged.

Implement:

state_hash = hash(all_driver_inputs)

If state_hash unchanged:

skip recalculation
return cached result

Log:

recalculation_skipped

--------------------------------

7) ENGINE INPUT VALIDATION

Before running forecast:

validate:

- driver_count
- driver_direction
- impact_score
- reliability_score
- normalization_factor

If any field missing:

reject forecast
display:

INVALID DRIVER INPUT

--------------------------------

8) DIAGNOSTIC PANEL

Add hidden debug panel showing:

Number of drivers
Duplicate drivers detected
Largest single shift
Total shift
Gating constraints triggered
Final probability limit reason

This panel is for system validation, not end users.

--------------------------------

SUCCESS CRITERIA

The system must never output near-certain probability when:

- key gating constraints remain unresolved
- relevance check indicates partial applicability
- adoption threshold feasibility is uncertain

Probability should reflect constraint reality, not signal strength alone.
