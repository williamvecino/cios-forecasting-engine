CRITICAL BUG: MANUAL SIGNALS ARE NOT PERSISTING OR OVERRIDING AI SIGNALS

PROBLEM

When the user manually adds a signal after seeing the forecast, the system ignores it or reverses its intended effect.

Example:
User adds a manual signal saying reimbursement/payer constraint is no longer needed.
Expected:
- Payer Coverage gate improves
- Final forecast improves
Actual:
- gate does not improve or gets worse
- manual signal disappears after navigating back
- Step 2 regenerates AI signals and overwrites manual work

This is a major state and precedence bug.

==================================================
REQUIRED FIXES
==================================================

1. MANUAL SIGNALS MUST PERSIST
Manual signals must be saved to case state immediately and persist across:
- Step 2 -> Step 3
- Step 3 -> Step 2
- Step 3 recalculation
- page refresh if possible

Do not store them only in temporary UI state.

2. DO NOT REGENERATE SIGNALS ON STEP NAVIGATION
Going back to Step 2 must reopen the existing saved signal set.
Do not rerun AI signal generation automatically when navigating between steps.

Only regenerate if the user explicitly clicks:
- Reanalyze
- Refresh signals
- Regenerate signals

3. MANUAL SIGNALS MUST HAVE PRIORITY
If the user manually enters a signal, it must take precedence over generic AI-generated uncertainty signals in the same domain.

Priority order:
manual_confirmed > observed_verified > ai_derived > ai_uncertainty

Add field:
signal_priority_source
Allowed values:
- manual_confirmed
- observed_verified
- ai_derived
- ai_uncertainty

4. ADD SIGNAL LOCKING
Manual signals should be locked by default from auto-overwrite.
AI regeneration must not delete or replace manual signals unless the user explicitly chooses that.

Add field:
is_locked
Default:
true for manual signals

5. FIX GATE POLARITY
A positive reimbursement/access signal must not worsen the reimbursement gate.

If signal says:
payer constraint resolved / access improved / reimbursement barrier reduced

Then:
Access/Reimbursement gate must improve, not decline.

Add a polarity validation check:
positive access signal cannot reduce access gate score

6. ADD SIGNAL CONFLICT RESOLUTION
If an AI signal says reimbursement uncertain but a manual signal says reimbursement barrier resolved, the system must:
- detect conflict
- show both
- favor manual_confirmed unless a stronger contradictory verified source exists

Do not silently average them without explanation.

7. ADD SCENARIO / WHAT-IF MODE
Manual signals added after seeing a forecast should be allowed to act as scenario inputs.

Add option:
Apply as scenario assumption

If checked:
- update gates
- update forecast
- mark output as scenario-adjusted

This is essential for scenario studies.

8. STEP STATE MUST BE STABLE
Moving between Step 2 and Step 3 must not mutate the case unless:
- user edits a signal
- user confirms regeneration
- user deletes a signal

Navigation alone must not trigger reanalysis.

==================================================
DATA MODEL CHANGES
==================================================

For each signal add:

- signal_id
- signal_priority_source
- is_locked
- created_by_user
- persists_across_steps
- is_scenario_override
- target_gate_id
- direction
- confidence
- contribution_points

==================================================
VALIDATION RULES
==================================================

1. If manual signal is added and accepted:
- it must appear when user returns to Step 2
- it must still exist after returning to Step 3

2. If manual signal positively affects reimbursement:
- reimbursement gate score must not decrease

3. If no explicit regeneration is requested:
- AI must not recreate or replace signal set

4. If manual signal conflicts with AI signal:
- conflict must be surfaced
- manual signal must not disappear

==================================================
UI REQUIREMENTS
==================================================

Add small tags to signals:
- Manual
- Locked
- Scenario Override

Add a warning if regeneration would overwrite unlocked AI signals only.
Do not overwrite locked manual signals.

Add an explicit button:
Refresh AI Signals

Only this button may regenerate AI suggestions.
