CIOS BACKEND TASK — FORECAST LEDGER

Context:
The core engine, UI, scenario simulation, and signal taxonomy are complete.
We are now implementing the Forecast Ledger component.

Objective:
Create a Forecast Ledger layer that records predictions and later compares them with real outcomes to measure calibration and reliability.

This must be a tracking/evaluation layer only.
It must not change forecasting behavior.

Scope:
Create a ForecastLedger module/table/model with the following fields:

- PredictionID
- CaseID
- StrategicQuestion
- ForecastProbability
- ForecastDate
- TimeHorizon
- ExpectedResolutionDate
- ActualOutcome
- ResolutionDate
- PredictionError
- CalibrationBucket

Field rules:

PredictionID
- unique identifier for each forecast record

CaseID
- link to the original case

StrategicQuestion
- copy from CaseHeader at time of forecast capture

ForecastProbability
- probability generated by the existing engine

ForecastDate
- date/time the forecast was recorded

TimeHorizon
- copied from case/question metadata

ExpectedResolutionDate
- estimated date when outcome should become knowable

ActualOutcome
- nullable until resolved
- when resolved:
  - 1 = event occurred
  - 0 = event did not occur

ResolutionDate
- nullable until resolved

PredictionError
- computed only after outcome is resolved
- absolute difference between ForecastProbability and ActualOutcome

CalibrationBucket
- auto-derived from ForecastProbability into 10-point ranges:
  - 0–10%
  - 10–20%
  - 20–30%
  ...
  - 90–100%

Implementation requirements:

1. Add the ForecastLedger schema/model/table.
2. Add a read-only capture mechanism so that when a forecast is generated, a ledger record can be created from the existing output.
3. Add a safe/manual way to update ActualOutcome and ResolutionDate later.
4. Automatically compute PredictionError once outcome is entered.
5. Automatically assign CalibrationBucket from the forecast probability.
6. Keep this module independent from the Bayesian engine.

Strict constraints:
Do NOT modify:
- Bayesian engine
- posterior probability calculations
- likelihood ratio computation
- calibration logic
- signal processing
- scenario simulation behavior
- UI layout
- existing forecast outputs

Deliverable:
Return:
1. exact files created or modified
2. ForecastLedger schema/model
3. how forecast capture is triggered
4. how outcome resolution is updated
5. confirmation that no core logic changed
6. confirmation that forecast outputs remain unchanged
